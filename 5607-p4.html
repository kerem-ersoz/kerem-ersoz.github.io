CSCI 5607 Project 4:
Real Time Game Programming


In this project, I implemented a maze game. The game is a first person game involving doors that can be unlocked using keys scattered throughout the map.

$ make
$ ./play maps/2.txt

In the game, the player is trapped in a dungeon in a maze with a number of locked doors. The player must find the magical teapots around the dungeon to unlock these doors. At one end of the dungeon is a magic china teapot that contains a genie. The goal of the game is to unlock all doors and reach the magic china teapot. Implementation wise, I based most of my approach on Prof Guy's solution posted in Lua. I had a hard time loading model verts and getting them to render properly, where my walls would get cut into slices would almost look like window blinds. I found that this was due to a calculation error. Overall though, I found that the starter code posted by Prof Guy and the wealth of OpenGL documentation online made things relatively straightforward to implement. Ultimately, there are a few minor things that I was unable to address: 1. I animated the opening of doors based on a tick value after the program has been started, meaning that after a while of running the game, door animations would appear to be instant. Secondly, I was unable to position the diffuse light source such that there wouldn't be large shadows in the map. I believe these were fairly minor though.
